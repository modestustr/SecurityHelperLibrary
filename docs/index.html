Multi-target cryptography

SecurityHelperLibrary

A security-first helper library that binds PBKDF2, Argon2id, HMAC, and AES-GCM while keeping architectural separation clean for both .NET Framework and .NET 8 projects.

Get Started NuGet Page

Highlights

Timing-Attack Resistance

Fixed-time comparisons and cryptographic helpers prevent leakage during verification.

`ReadOnlySpan` overloads and `ClearSensitiveData` call sites keep secrets off long-lived memory.

Modern + Legacy Targets

Single package works for both `net481` and `net8.0`, each with dedicated security gates.

Release 2.1.2 added RBAC auditing, incident telemetry, and hardened error surfaces.

Argon2 Safeguards

Minimum 64MB memory, capped parallelism, and enforced iteration depth balance safety vs resources.

Non-supported targets throw safe `NotSupportedException` wrappers instead of failing silently.

AES-GCM + HMAC

Convenient encrypt/decrypt helpers and HMAC utilities for modern runtimes.

Key creation, IV management, and cleanup helpers ship with zeroing utilities.

Quick Start

PBKDF2 Password

SHA-256, 210,000 iterations, and a 32-byte hash for high-cost verification.

var helper = new SecurityHelper();
var stored = helper.HashPasswordWithPBKDF2("MyPassword123!", out var salt, HashAlgorithmName.SHA256, iterations: 210000);
var valid = helper.VerifyPasswordWithPBKDF2("MyPassword123!", stored);

Use the `stored` payload anywhere persistent storage is needed.

Argon2id

Four iterations, 128MB memory, and parallelism of 4 for modern hash safety.

string salt = helper.GenerateSalt();
string hash = helper.HashPasswordWithArgon2("MyPassword123!", salt, iterations: 4, memoryKb: 131072, degreeOfParallelism: 4, hashLength: 32);
bool verified = helper.VerifyPasswordWithArgon2("MyPassword123!", hash);

Store the hash and salt, then verify when needed.

AES-GCM

Encrypt strings and protect secrets with AES-GCM on supported targets.

byte[] key = helper.GenerateSymmetricKey(32);
string cipher = helper.EncryptStringGCM("sensitive value", key);
string plain = helper.DecryptStringGCM(cipher, key);
helper.ClearSensitiveData(key);

Always clear buffers when done to keep secrets from lingering.

Release Flow & Pipelines

Every release follows a `release/x.y.z` branch, runs `./scripts/release.bat`, and updates `CHANGELOG.md` plus `` metadata. Pentest and build checks (GitHub Actions) must pass before merging into master.

Checkpoints

  • Master is locked until branch protection, pentest, and build success are verified.
  • Security incidents log Ketum codes internally while exposing generic errors externally.
  • Release branches auto-bump versions and regenerate NuGet metadata.

Tests

  • `dotnet test` covers every target with the full xUnit suite.
  • The pentest pack runs each commit against 13 attack vectors.
  • The sample app demonstrates JWT + role-based incident auditing.

Security Tips

Memory Hygiene

Always zero Argon2 and PBKDF2 outputs via `ClearSensitiveData`.

`ReadOnlySpan` overloads keep sensitive values transient and short-lived.

Logging & Telemetry

Internal logs map incidents to Ketum codes while external clients receive sanitized error messages.

Role-based auditing keeps `SecurityAuditController` dashboards viewable only by approved staff.
